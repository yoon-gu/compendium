에이전틱 루프, 컨텍스트 로딩, 도구 실행 모델, 그리고 내부에서 이루어지는 대화 생명주기에 대해 알아봅니다.

Claude Code는 터미널 기반 코딩 에이전트로, 연속적인 에이전틱 루프를 실행합니다. 사용자의 요청을 읽고, 무엇을 해야 할지 추론하고, 도구를 호출하고, 결과를 관찰한 뒤, 작업이 완료되거나 사용자의 입력이 필요할 때까지 이 과정을 반복합니다.

---

## 에이전틱 루프

모든 상호작용은 동일한 기본 사이클을 따릅니다:

**1단계 — 사용자가 메시지를 보냄**

터미널에서 메시지를 입력하거나(인터랙티브 모드), `--print` / stdin을 통해 전달합니다(비인터랙티브 / 헤드리스 모드). 메시지는 대화 기록에 추가됩니다.

**2단계 — 컨텍스트 조립**

모델을 호출하기 전에, Claude Code는 시스템 프롬프트를 조립합니다. 여기에는 현재 날짜, git 상태(브랜치, 최근 커밋, 작업 트리 상태), 로드된 CLAUDE.md 메모리 파일, 사용 가능한 도구 목록이 포함됩니다. 이 컨텍스트는 대화당 한 번 빌드되고 메모이제이션됩니다 — `context.ts` 참조.

**3단계 — Claude가 추론하고 도구를 선택**

조립된 대화가 Anthropic API로 전송됩니다. 모델은 작업에 대해 추론하고 하나 이상의 `tool_use` 블록을 출력합니다 — 각각 도구 이름과 구조화된 JSON 입력을 지정합니다.

**4단계 — 권한 확인**

각 도구 호출을 실행하기 전에, Claude Code는 현재 권한 모드와 허용/거부 규칙을 평가합니다. 모드에 따라 자동 승인하거나, 확인을 요청하거나, 호출을 완전히 차단합니다. 권한(Permissions) 문서를 참조하세요.

**5단계 — 도구가 실행되고 결과를 반환**

승인된 도구 호출이 실행됩니다. 결과 — 파일 내용, 명령어 출력, 검색 결과 등 — 는 `tool_result` 블록으로 대화에 추가됩니다.

**6단계 — 루프 계속**

모델은 도구 결과를 받아 추가 도구를 호출하거나 최종 텍스트 응답을 생성합니다. 모델 턴에서 도구 호출이 남아있지 않을 때까지 루프가 반복됩니다.

> 루프는 전적으로 사용자의 터미널 프로세스에서 실행됩니다. 원격 실행 서버가 없으며 — 파일, 셸, 자격 증명은 도구가 명시적으로 전송하지 않는 한(예: `WebFetch`, `WebSearch`, 또는 MCP 서버) 절대로 사용자의 머신을 떠나지 않습니다.

## 컨텍스트 로딩

각 대화의 시작 시, Claude Code는 모든 API 호출 앞에 추가되는 두 가지 컨텍스트 블록을 빌드합니다:

### 시스템 컨텍스트

`context.ts`의 `getSystemContext()`에서 조립됩니다. 포함 내용:

- **Git 상태** — 현재 브랜치, 기본/메인 브랜치, git 사용자명, `git status --short` 출력(2,000자 초과 시 잘림), `git log --oneline`의 최근 5개 커밋.
- **캐시 무효화 주입** — 디버깅 시 서버 측 프롬프트 캐시를 무효화하는 데 사용되는 선택적 임시 문자열.

`CLAUDE_CODE_REMOTE=1`이 설정되어 있거나(원격/클라우드 모드) 설정에서 git 지침이 비활성화된 경우 Git 상태를 건너뜁니다.

### 사용자 컨텍스트

`context.ts`의 `getUserContext()`에서 조립됩니다. 포함 내용:

- **CLAUDE.md 메모리** — 4단계 계층(관리형 → 사용자 → 프로젝트 → 로컬)에서 발견된 모든 메모리 파일. `CLAUDE_CODE_DISABLE_CLAUDE_MDS=1`이나 명시적 `--add-dir` 없는 베어 모드에서 비활성화됩니다. 메모리 & 컨텍스트 문서를 참조하세요.
- **현재 날짜** — 모델이 항상 날짜를 알 수 있도록 `Today's date is YYYY-MM-DD` 형식으로 주입됩니다.

두 컨텍스트 블록 모두 `lodash/memoize`를 사용하여 대화 기간 동안 **메모이제이션**됩니다. `setSystemPromptInjection()`을 호출하면 캐시가 즉시 초기화됩니다.

## 도구 실행 모델

Claude Code는 기본적으로 도구 호출을 자율적으로 실행하지 않습니다. 각 도구에는 `checkPermissions` 메서드가 있으며, 그 결과에 따라 다음 동작이 결정됩니다:

| 권한 결과 | 동작 |
|-----------|------|
| `allow` | 도구가 즉시 실행되고, 결과가 대화에 추가됨 |
| `ask` | Claude Code가 일시 정지하고 확인 대화상자를 표시 |
| `deny` | 도구 호출이 거부되고, Claude에 에러 결과가 전달됨 |

권한 동작은 활성 권한 모드와 구성된 허용/거부 규칙에 의해 제어됩니다. `bypassPermissions` 모드에서는 모든 검사를 건너뜁니다. `acceptEdits` 모드에서는 파일 편집 도구가 자동 승인되지만 bash 명령은 여전히 확인을 요청합니다.

안전하고 읽기 전용인 도구 호출(예: `Read`, `Glob`, `Grep`)은 일반적으로 모든 모드에서 자동 승인됩니다.

## 인터랙티브 vs. 비인터랙티브(태스크) 모드

### 인터랙티브(REPL) 모드

기본 경험입니다. Claude Code는 React/Ink를 사용하여 라이브 터미널 UI를 렌더링합니다. 에이전트가 작업하는 동안 스트리밍 출력, 도구 사용 확인, 스피너 애니메이션을 볼 수 있습니다. 메시지는 종료할 때까지 세션 전체에서 유지됩니다.

### 비인터랙티브 / print 모드

`--print`나 stdin 파이핑으로 활성화됩니다. UI가 렌더링되지 않습니다. 출력은 stdout으로 기록되어 스크립트나 CI 파이프라인에서 캡처할 수 있습니다. 일회성 자동화 작업에 유용합니다.

### 서브 에이전트 (Task 도구)

Claude는 `Task` 도구(`AgentTool`)를 통해 서브 에이전트를 생성할 수 있습니다. 각 서브 에이전트는 격리된 대화와 선택적으로 제한된 도구 집합을 가진 자체 중첩 에이전틱 루프를 실행합니다. 서브 에이전트는 로컬(인프로세스) 또는 원격 컴퓨팅에서 실행할 수 있습니다. 서브 에이전트가 완료되면 결과가 부모 에이전트로 반환됩니다. 자세한 내용은 도구(Tools) 문서를 참조하세요.

## 대화 저장 및 재개

대화는 디스크의 JSON 트랜스크립트 파일로 저장됩니다(기본 경로: `~/.claude/`). 각 대화에는 고유한 세션 ID가 있습니다. `--resume <session-id>`로 이전 대화를 재개하거나, `--resume`만으로 목록에서 선택할 수 있습니다.

대화가 재개될 때:

- 전체 메시지 기록이 디스크에서 로드됩니다.
- 메모리 파일이 다시 탐색되며, 대화를 처음 시작했을 때와 다를 수 있습니다.
- 권한 모드는 세션에 유지되지 않은 경우 구성된 기본값으로 재설정됩니다.

긴 대화는 주기적으로 **압축**됩니다 — 가장 오래된 메시지가 요약되어 컨텍스트 윈도우를 관리 가능한 수준으로 유지합니다. 전체 원본 트랜스크립트는 항상 디스크에 보존됩니다. 압축은 API로 전송되는 내용에만 영향을 미칩니다.

## 쿼리 엔진

내부적으로, 에이전틱 루프의 각 "턴"은 **쿼리**에 의해 구동됩니다 — 현재 메시지 목록을 Anthropic API로 보내고 응답을 스트리밍으로 받는 `query.ts` 호출입니다. 쿼리 엔진이 처리하는 내용:

- 터미널에 실시간으로 토큰 출력 스트리밍.
- `tool_use` 블록을 적절한 도구 핸들러로 디스패치.
- 턴당 토큰 및 도구 호출 예산 적용.
- 도구 결과를 수집하고 다음 모델 호출 전에 추가.
- 컨텍스트 윈도우가 가득 차면 압축 트리거.

각 도구에는 `maxResultSizeChars` 속성이 있습니다. 결과가 이 제한을 초과하면 내용이 임시 파일에 저장되고 모델은 파일 경로가 포함된 미리보기를 받아, 대용량 출력으로 인한 컨텍스트 윈도우 오버플로를 방지합니다.
