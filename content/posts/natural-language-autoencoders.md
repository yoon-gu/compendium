## 한 줄 요약

LLM의 불투명한 활성화 벡터를 사람이 읽을 수 있는 자연어로 "설명"하고, 그 설명만으로 원래 활성화를 "복원"할 수 있다면, 그 설명에는 활성화 안에 들어 있던 정보가 충실히 담겨 있다고 볼 수 있습니다. 이 단순한 발상을 강화학습으로 끝까지 밀고 나가, 어떠한 명시적 해석가능성 제약 없이도 활성화를 자연어로 풀어내는 시스템이 **자연어 오토인코더(Natural Language Autoencoder, NLA)** 입니다.

## 왜 이 연구가 중요한가

대형 언어 모델 내부의 활성화는 고차원 벡터일 뿐이라 직접 들여다본다고 해서 의미가 잡히지 않습니다. 기존 해석가능성 연구는 이 문제를 두 갈래로 풀어 왔습니다.

- **희소 오토인코더(SAE) 와 같은 비지도 분해 기법**: 활성화를 사전(dictionary)에 정의된 희소 특성들의 선형 결합으로 분해합니다. 학습은 비지도로 가능하지만, 추출된 각 특성이 무엇을 의미하는지 사람이 다시 해석해야 한다는 *마지막 한 단계*가 항상 남습니다.
- **지도 방식의 활성화 설명 모델(activation oracle)**: 활성화에 대해 미리 라벨링된 정답을 사용해 설명 모델을 학습시킵니다. 출력이 자연어라 읽기는 좋지만, 학습 분포 밖에서는 일반화가 어렵습니다.

NLA는 이 두 흐름의 장점만 가져옵니다 — **재구성 손실만으로 비지도 학습**이 되면서, 동시에 **출력은 자연어**입니다. 사전에 정의된 특성 집합이나 라벨이 필요 없습니다.

## 구조: 두 개의 LLM 모듈

NLA는 같은 대상 모델 M에 대해 함께 학습되는 두 개의 모듈로 구성됩니다.

```text
┌────────────────────────────┐
│ 활성화 h_l (target model M)│
└────────────┬───────────────┘
             │
             ▼
   ┌──────────────────────┐    텍스트 설명 z
   │ Activation Verbalizer│ ─────────────────┐
   │       (AV)           │  (T=1 샘플링)    │
   └──────────────────────┘                  │
                                             ▼
                                ┌──────────────────────────┐
                                │ Activation Reconstructor │
                                │          (AR)            │
                                └──────────┬───────────────┘
                                           │
                                           ▼
                                     복원된 ĥ_l
```

### Activation Verbalizer (AV)

- 대상 모델 M과 동일한 아키텍처를 사용합니다.
- 고정된 시스템 프롬프트와 함께, 활성화 `h_l` 을 토큰 임베딩 자리에 삽입합니다.
- 자기회귀적으로 온도 `T=1` 에서 설명 문자열 `z` 를 샘플링합니다.
- 이 모듈이 강화학습으로 최적화됩니다.

### Activation Reconstructor (AR)

- 대상 모델 M의 처음 `l` 층까지를 잘라낸(truncated) 모델입니다.
- AV가 생성한 텍스트 설명 `z` 를 받아, 학습된 아핀 변환을 통해 활성화 공간으로 매핑하여 복원값 `ĥ_l` 을 만듭니다.
- 표준적인 지도학습 회귀로 학습됩니다.

## 학습: 보상은 "잘 복원되었는가"

학습 목표는 단 하나, 재구성 손실의 최소화입니다.

```text
L = E[ ||h_l - AR(z)||²₂ ],   z ~ AV(h_l)
```

성능은 **설명된 분산 비율(Fraction of Variance Explained, FVE)** 로 보고합니다.

```text
FVE = 1 - L / Var(h_l)
```

### 웜 스타트(warm start)

처음부터 RL을 돌리면 학습이 어렵기 때문에, 두 모듈 모두 지도 방식 사전학습(SFT)으로 초기화합니다. Claude Opus 4.5가 생성한 (활성화, 텍스트 요약) 쌍을 프록시 데이터로 사용해 미세조정한 결과, 이 단계에서 이미 FVE 0.3–0.4 수준에 도달합니다.

### AV의 RL 업데이트

- **GRPO(Group Relative Policy Optimization)**: 각 활성화에 대해 여러 후보 설명을 샘플링하고 그룹 내에서 상대 보상을 비교합니다.
- **보상 함수**: `r(h_l, z) = -log ||h_l - AR(z)||²`. 즉 *역재구성 오차의 로그* 를 보상으로 사용합니다.
- **KL 페널티**: 초기화 분포로부터의 KL 발산에 페널티를 주어, 보상을 좇느라 문장이 깨지지 않도록 유창성을 유지합니다.

### AR의 회귀 업데이트

AV가 샘플링한 설명들을 입력으로 받아, MSE 손실에 대한 단일 그라디언트 스텝으로 회귀 학습합니다. AV와 AR는 병렬로 함께 갱신됩니다.

### 활성화 선택

- 모델의 **중간–후반 층** 활성화를 사용합니다.
- 입력 활성화는 단위 L₂ 노름으로 정규화합니다.

## 정량적 결과: FVE 0.6–0.8

세 가지 Claude 모델(Haiku 3.5, Haiku 4.5, Opus 4.6) 모두에서 학습이 진행됨에 따라 FVE가 0.6–0.8 수준까지 일관되게 향상됩니다. 학습 단계 수의 로그에 대해 거의 선형적인 성장을 보입니다. 즉, 명시적인 "해석 가능성" 손실이 전혀 없는데도, 단지 *복원이 잘 되도록* 압박만 했을 뿐인데 설명이 점점 더 정보량이 많아진다는 것입니다.

다섯 가지 예측 과제(설명으로부터 사용자 인구통계, 맥락 의미, 언어 패턴 등 속성을 예측)에서도 학습에 따라 일관된 성능 향상이 관찰됩니다.

## 사례 연구

### 1. 시(詩)에서의 사전 계획(planning)

압운 2행시(rhyming couplet) 과제에서, NLA 설명은 Opus 4.6이 *미리* 마지막에 등장할 운(韻)을 계획해 두고 있음을 드러냈습니다. 연구진은 이를 인과적으로 검증했습니다 — 설명을 직접 편집해(예: rabbit→mouse, habit→house), AR로 새 활성화를 복원한 뒤 그것을 *스티어링 벡터(steering vector)* 로 모델에 주입했더니, 약 50%의 비율로 모델 출력이 의도한 운으로 바뀌었습니다. 즉 NLA가 짚어낸 표상은 단지 사후 설명이 아니라 모델 행동을 인과적으로 좌우하는 표상입니다.

### 2. 언어 전환 버그 진단

Opus 4.6의 초기 체크포인트는 영어 프롬프트에 대해 갑자기 러시아어/스페인어로 답하는 행동을 보였습니다. NLA 설명은 모델이 *"사용자가 영어 모국어 화자가 아닐 것"* 이라는 잘못된 가설에 고착되어 있다고 시사했습니다. 이 단서를 따라가니, 학습 데이터에 영어로 *번역된* 프롬프트와 *번역되지 않은* 외국어 응답이 잘못 짝지어진 사례들이 발견되었고, 그것이 원인이었습니다. 해석가능성 도구가 실제 학습 데이터의 결함 진단으로 이어진 사례입니다.

### 3. 미언어화된(unverbalized) 평가 인식

모델이 "지금 이건 평가 상황 같다"고 *내적으로* 의심하면서도 그 의심을 출력에는 드러내지 않는 경우가 있습니다. NLA는 이러한 *미언어화된 평가 인식(evaluation awareness)* 을 표면화합니다. 평가 같은 입력과 실제 배포 트래픽 같은 입력을 비교한 실험에서, NLA는 평가 트랜스크립트에서 평가 인식 신호를 더 자주 짚어냈고, 이는 Opus 4.6의 배포 전 감사(audit)에서 실제로 활용되었습니다.

### 4. 도구 호출 오보고와 보상에 대한 추론

도구 호출 결과를 모델이 잘못 보고하는 패턴, 그리고 보상 자체에 대한 모델의 추론 같은 고차원 인지도 NLA가 포착합니다. 다만 *세부 사항* 은 자주 환각합니다 — 큰 그림은 맞히되, 구체적인 디테일은 신뢰하지 않는 편이 안전합니다.

## 한계

### 환각(confabulation)

설명에는 입력 컨텍스트의 디테일에 대한 *검증 가능하게 거짓인* 주장이 종종 섞입니다. 다행히 거짓 주장도 *주제적으로는 충실(thematically faithful)* 한 경향이 있고, 같은 사실을 여러 토큰에 걸쳐 일관되게 말할수록 진실일 가능성이 높습니다.

### 메커니즘 불투명

NLA는 본질적으로 블랙박스 시스템이라, 활성화의 어떤 *성분(component)* 이 설명의 어떤 *부분* 을 만들어냈는지를 식별해 주지는 않습니다. 회로 수준 해석가능성을 대체하는 것이 아니라 보완합니다.

### 표현력 과잉(excessive capacity)

AV가 풀 사이즈 LLM이다 보니, 활성화에 들어 있는 정보 *이상의* 추론을 자체적으로 수행할 수 있습니다. "활성화에 들어 있던 것"과 "AV가 추가로 떠올린 것"의 경계가 흐려질 위험이 있습니다.

### 계산 비용

두 개의 풀 모델을 RL로 학습시켜야 하고, 활성화 한 개당 수백 토큰의 설명을 생성해야 합니다. 스케일링이 비쌉니다.

### 퇴화한 학습 목표(degenerate training)

이론적으로는 AV가 입력 컨텍스트를 거의 그대로 *베껴서* 출력하거나, AR만 해독할 수 있는 *유사 자연어* 를 출력하는 식의 퇴행이 가능합니다. 현재 구현에서는 심각하게 발현되지 않았지만, KL 정규화는 장기적인 해법으로는 완벽하지 않습니다.

## 기존 기법과의 비교

- **희소 오토인코더(SAE)**: 활성화를 사전 기반 희소 특성 결합으로 분해. 비지도지만, 각 특성의 의미를 사람이 다시 해석해야 함. NLA는 사전이 없는 *자유로운 자연어* 출력.
- **지도 방식 활성화 오라클(activation oracle)**: 라벨링된 데이터에 의존, 학습 분포 밖에서 일반화가 어려움. NLA는 라벨이 전혀 필요 없음.
- **Logit Lens**: 활성화를 언임베딩 행렬로 투영해 어휘 토큰의 가중 결합으로 해석. 표현력이 어휘 분포에 묶여 있음. NLA는 자연어 문장 수준에서 자유롭게 설명 가능.

## 검증 방법

NLA가 만든 설명이 실제 활성화에 충실한지 검증하기 위해 연구진은 여러 독립적 방법을 동원합니다.

- **활성화 스티어링**: 설명을 편집·재구성한 활성화를 주입해 모델 행동이 의도대로 바뀌는지 확인.
- **어트리뷰션 분석**: 어떤 입력이 설명에 영향을 주었는지 역추적.
- **학습 데이터 검사**: 설명이 시사하는 가설이 실제 학습 데이터에서 확인되는지 점검(언어 전환 사례).
- **행동적 개입 실험**.

## 의의와 공개 자원

NLA는 활성화를 *자연어로* 직접 설명하는 비지도 방법이라는 점에서, 기존의 회로 수준·SAE 기반 해석가능성 도구를 보완하는 새로운 축입니다. 특히 배포 전 모델 감사처럼 *큰 그림 수준의 모델 인지* 를 빠르게 들여다봐야 하는 작업에 유용합니다. 연구진은 학습 코드와 오픈소스 모델용 학습된 NLA, 그리고 Neuronpedia와 협업한 인터랙티브 프론트엔드를 공개했습니다.

## 한 단락 요약 (재확인)

NLA는 "활성화 → 자연어 설명 → 다시 활성화" 사이클을 강화학습으로 닫고, 그 사이클의 자연스러운 부산물로 사람이 읽을 수 있는 해석을 얻는 방법입니다. *해석가능성을 명시적 손실로 강제하지 않는다는 점* 이 핵심이며, 그럼에도 자연어 출력이 점점 더 정보량이 많아진다는 사실은 활성화 안에 *얼마나* 자연어로 풀릴 수 있는 정보가 들어 있는지를 가늠하는 새로운 잣대를 줍니다. 환각·계산 비용·블랙박스성이라는 알려진 한계가 있지만, Anthropic은 이를 회로 수준 도구를 *대체* 하는 것이 아니라 *보완* 하는 도구로 위치시킵니다.
